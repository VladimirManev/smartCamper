# Module-5: Контролер за Консуматори

ESP32 модул за управление на шест консуматора чрез релета: аудиосистема, помпа за вода, хладилник, вентилатор в тоалетна, бойлер и инвертор.

### Функции на бутоните

- **Клик**: Превключване реле ON/OFF

## Мрежова конфигурация

- **WiFi SSID**: `SmartCamper`
- **WiFi Парола**: `12344321`
- **MQTT Broker IP**: `192.168.4.1` (Raspberry Pi)
- **MQTT Broker Порт**: `1883`
- **ID на модула**: `module-5`
- **MQTT Buffer Size**: 1024 байта

## MQTT Topics

### Публикувани (Статус)

| Topic                                 | Формат на съобщението                                                        | Честота на обновяване                                             |
| ------------------------------------- | ---------------------------------------------------------------------------- | ----------------------------------------------------------------- |
| `smartcamper/sensors/module-5/status` | `{"relays": {"0": {"state": "ON"}, ...}}`                                    | Само при промяна (натискане на бутон, MQTT команда, force_update) |
| `smartcamper/sensors/toilet/urine/level` | `50` (0 / 50 / 100)                                                      | При промяна (≥1%) или `force_update`                              |
| `smartcamper/heartbeat/module-5`      | `{"timestamp": ..., "moduleId": "module-5", "uptime": ..., "wifiRSSI": ...}` | На всеки 10 секунди                                               |

### Абонирани (Команди)

| Topic                                                | Payload | Действие                          |
| ---------------------------------------------------- | ------- | --------------------------------- |
| `smartcamper/commands/module-5/relay/{index}/toggle` | `{}`    | Превключване реле                 |
| `smartcamper/commands/module-5/force_update`         | `{}`    | Принудително обновяване на статус |

### Формат на командите

- `{index}` може да бъде 0, 1, 2, 3, 4 или 5 (за шестте релета)
- Пример: `smartcamper/commands/module-5/relay/0/toggle` - превключва реле 0 (Аудиосистема)
- Пример: `smartcamper/commands/module-5/relay/3/toggle` - превключва реле 3 (Вентилатор WC)
- Пример: `smartcamper/commands/module-5/relay/4/toggle` - превключва реле 4 (Бойлер)
- Пример: `smartcamper/commands/module-5/relay/5/toggle` - превключва реле 5 (Инвертор)

## Функционалности

### Управление на релета

- Всяко реле може да се управлява независимо
- Физическите бутони работят офлайн (без WiFi/MQTT)
- MQTT командите работят когато модулът е свързан
- Статусът се публикува автоматично при всяка промяна

### Поведение

- Същото поведение като Ambient релето в module-2
- Просто toggle - няма димиране или сложни режими
- Бутонът превключва релето веднага при натискане

### Сензор за ниво на урина (тоалетна)

Два проводими електрода в уринения резервоар (същата логика като сивата вода в module-1, по-малко нива):

| GPIO | Ниво |
| ---- | ---- |
| 4 | 50% |
| 5 | 100% |

- Интервал на четене: 30 s
- Публикувано ниво: **мода** от последните 10 проби (~5 мин), или последно четене при `force_update` / reconnect
- Възможни стойности: **0%, 50%, 100%**

Конфигурация: `URINE_LEVEL_*` в `src/Config.h`.

## Backend

- **Релета**: `module-5/status` → `applianceStatusUpdate`
- **Урина**: `toilet/urine/level` → `sensorUpdate` с `{ toiletUrineLevel }`
- **Force refresh**: `forceModuleUpdate` с `{ moduleId: "module-5" }`

## Frontend

- **Карта Toilet** в менюто — жълта течност, надпис **WC**
- **Модал** — poll на module-5 на всеки 5 s докато е отворен
- **Status панел** (tablet) — в ротацията на слайдовете

## Инсталация и настройка

1. **Качване на firmware:**

   ```bash
   cd esp32-modules/module-5
   pio run --target upload
   ```

2. **Мониторинг на serial изход:**

   ```bash

   ```

   - Проверка на WiFi/MQTT връзка
   - Мониторинг на натискания на бутони
   - Проверка на статус на релета

3. **Проверка на работата:**
   - Тест на физическите бутони
   - Проверка на frontend dashboard
   - Проверка на MQTT команди

## Отстраняване на проблеми

### Релетата не работят

- Проверете захранването на релетата (12V DC)
- Проверете свързванията на релетата (пинове 14, 15, 16, 23, 27, 32)
- Проверете serial изхода за грешки
- Уверете се в правилно ground свързване между ESP32 и релетата

### Бутоните не работят

- Проверете свързването на бутоните (пинове 17, 21, 22, 25, 26, 33)
- Проверете pull-up резисторите (използват се вътрешни pull-up-и)
- Тест с мултиметър (бутонът трябва да свързва пин към GND при натискане)
- Проверете serial изхода за съобщения за натискане

### Няма MQTT връзка

- Проверете IP адреса на MQTT broker в `src/Config.h`
- Проверете статуса на WiFi връзката
- Уверете се, че MQTT broker работи на Raspberry Pi
- Проверете serial изхода за MQTT грешки

### Грешно ниво на урина

- Проверете електродите GPIO 4 и 5, общ GND в резервоара
- Нивото е дискретно (0 / 50 / 100), не непрекъснато
- Изчакайте mode прозореца (~5 мин) или изпратете `force_update`

## Офлайн работа

Модулът работи независимо:

- Физическите бутони функционират без WiFi/MQTT
- Релетата могат да се управляват чрез бутони офлайн
- Обновленията на статус се публикуват само при свързаност
- Няма грешки когато е офлайн - всички локални функции работят

## Електрически спецификации

- **Релета**: Стандартен 12V реле модул (NO/NC контакти)
- **Консумация на ток**: Зависи от консуматорите (аудиосистема, помпа, хладилник)
- **Захранване**: 12V DC (достатъчен капацитет за всички консуматори)
- **Бутони**: Моментарни натискащи бутони (нормално отворени)

## Архитектура

- **ModuleManager**: WiFi, MQTT, Heartbeat, Команди
- **ApplianceManager**: Координира релета, бутони, уринен сензор, команди
- **RelayController**: Управлява релетата (контрол, статус)
- **ButtonHandler**: Обработва входове от бутони (debouncing, state machine)
- **UrineLevelSensor**: Ниво в тоалетната чрез проводими електроди

## Serial Debug Изход

Включване/изключване в `src/Config.h`:

- `DEBUG_SERIAL`: Serial конзолен изход
- `DEBUG_MQTT`: MQTT съобщения за публикуване/абониране
- `DEBUG_VERBOSE`: Подробни промени в състоянието на релетата
